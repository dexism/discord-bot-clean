# 開発参加ガイド (Contributing Guide)

プロジェクトへの貢献に関心を持っていただきありがとうございます。
このガイドは、開発者がスムーズにプロジェクトに参加し、貢献できるようにするためのものです。

## 🤝 貢献の方法

### 1. Issueの確認と作成
* 既知のバグや機能要望は [Issues](../../issues) で管理されています。
* 新しいバグを見つけた場合や、新機能を提案したい場合は、適切なテンプレート（Bug report / Feature request）を使用してIssueを作成してください。

### 2. 環境構築
`README.md` の手順に従い、ローカル環境をセットアップしてください。
基本的なコマンド:
```bash
npm install
npm run dev  # 開発サーバー起動
npm test     # テスト実行
```

### 3. ブランチ戦略
* `main` ブランチ: 常にデプロイ可能な最新の安定版
* `feature/xxx`: 新機能開発用
* `fix/xxx`: バグ修正用

### 4. Pull Request (PR) の作成
* 実装が完了したら、`main` ブランチに向けてPRを作成してください。
* PRの際のチェックリスト:
  - [ ] `npm run build` がエラーなく通るか
  - [ ] `npm test` がパスするか
  - [ ] 関連するドキュメント（もしあれば）を更新したか

## 📐 コーディング規約
* **言語**: TypeScript (Strict mode) を使用します。
* **フォーマット**: 基本的に既存のコードスタイル（インデント4スペース等）に従ってください。
* **命名**: 変数名は分かりやすい英語を使用してください。
* **非同期処理**: `async/await` を推奨します。
